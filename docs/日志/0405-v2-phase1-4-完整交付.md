### 过程中遇到的坑

**1. Studio UI 第一版设计被否决**
- 直接用 @uiw/react-md-editor 导致暗色代码编辑器风格
- 和 AutoMedia 整体暖色调、卡片化设计完全脱节
- 洋哥直接说"太丑了"
- 学到：功能页面设计必须和整体风格一致，"在纸上写字"vs"在 IDE 写代码"
- 重做：改用原生 textarea 定制样式，参考小红书/公众号发布页

**2. Next.js Turbopack 原生模块兼容**
- `@resvg/resvg-js` 是 native module，Turbopack 打包报错
- 修复：加入 `serverExternalPackages`

**3. 分享卡片字体问题**
- satori 不支持 variable font 和 TTC 格式
- macOS 系统字体都是 TTC（PingFang.ttc、STHeiti Light.ttc）
- Google Noto Sans SC 默认下载的也是 variable font
- 修复：使用 Adobe Source Han Sans SC 静态 OTF
- 额外：resvg 默认会扫描系统字体，发现 TTC 就报错，需要 `loadSystemFonts: false`

**4. 实体提取 "Too many parameter values" bug**
- 修复前：schema 没有 article_index，实体 fan-out 到整批 10 篇文章
- Codex 发现：mention_count 重复计数膨胀
- 修复后：schema 加 article_index，insertEntity 参数对齐，UNIQUE 索引防重复
