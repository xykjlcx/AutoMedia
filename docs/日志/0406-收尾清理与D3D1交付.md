# 0406 收尾清理与 D3/D1 交付

## 收尾清理（6 个原子提交）

### RSS 采集加重试
- 问题：知乎等 RSSHub 公共实例偶发 503，采集直接失败
- 方案：`rss.ts` 加 1 次重试，5xx 或网络错误时等 2s 再试，重试时重建 AbortSignal
- 耗时：~10min

### 禁用无效源
- Shopify Blog RSS `https://www.shopify.com/blog/feed` 返回 404，catalog 里替换为 `shopify.engineering/blog.atom`
- wechat 源 type='private' 无对应 collector，migration 自动 disable
- 用 `eq(sourceConfigs.rssUrl, ...)` 和 `eq(sourceConfigs.type, 'private')` 匹配，不依赖硬编码 ID

### 设置页 bug
- `TYPE_LABELS` 缺 `xiaohongshu-private`，源列表里小红书不显示类型徽章 → 加上红色标签
- maxItems 输入框 `max={10}` 但实际数据是 20，改为 `max={100}`

### 日报卡片来源缺失
- 问题：`DigestCard` 从静态 `SOURCE_META` 读不到 `custom-*` 源名
- 原因：`digest-page.tsx` 动态加载了源映射，但没传给 `DigestCard`
- 方案：`DigestCard` 新增 `sourceMeta` prop，父组件传动态映射，回退静态常量

## D3 语气润色

### 架构
- `POST /api/studio/polish` — 接收 text/tone/platform/context，用 `generateText` + fast model 改写
- `PolishToolbar` 组件 — 4 个按钮（口语化/正式/精简/扩展），调 API 后回调 `onPolished`
- `DraftEditor` — 监听 `mouseup`/`keyup` 检测选中，mirror div 计算坐标，渲染工具栏

### 关键决策
- 不用富文本编辑器，textarea 原生 `selectionStart/End` 足够精确
- 工具栏定位：用隐藏 mirror div 复制 textarea 样式测量光标位置，`onBlur` 延迟 200ms 关闭（避免点击按钮时工具栏消失）
- `onMouseDown={e => e.preventDefault()}` 在工具栏上阻止 blur 事件
- 最小选中 4 字触发，避免误操作
- 耗时：~30min（含 API + 组件 + 集成）
