### Spec 2：Studio 写作体验补齐

**Spec 文件：** `docs/superpowers/specs/2026-04-06-spec2-studio-writing-upgrade-design.md`

#### D2 草稿版本历史

AI 生成前后自动 snapshot（`pre_regenerate` + `ai_generate`），用户也能手动存快照（`manual_save`）。回滚前自动 snapshot 当前内容防止误操作。每 draft 保留最多 20 版本，满了先删非 `manual_save` 的。

- **新表：** `draft_versions`（draftId + source + createdAt）
- **新文件：** `src/lib/studio/versions.ts`、`src/app/api/studio/drafts/[id]/versions/*`、`src/components/studio/draft-versions.tsx`

- **保留与 `aiOriginal` 共存：** aiOriginal 继续用于风格学习 diff，version 是人工回滚历史，两者目的不同

#### D4 Studio 预览升级

把老的 `simpleMarkdownRender`（10 行正则）替换为真正的 markdown 渲染器：`react-markdown` + `remark-gfm`（表格/任务列表/URL 自动链接）+ `rehype-highlight`（代码块高亮）。

- **新依赖：** `react-markdown` `remark-gfm` `rehype-highlight` `highlight.js`（合计 ~100KB gzip）
- **新组件：** `src/components/studio/markdown-preview.tsx`（封装预览）
- **删除：** studio-page.tsx 里原来的 `simpleMarkdownRender` 函数
- **不引入 `rehype-raw`**：拒绝嵌入原始 HTML，避免 XSS 风险

#### D5 图文配图生成

在 Studio 底部新增「配图」按钮（条件性显示：配了 image provider 才显示）。点击弹出对话框，输入 prompt + 选比例 → 调用 Gemini 或 OpenAI image API → 生成 PNG 存到 `public/images/generated/` → 返回 markdown 链接可复制粘贴。

- **ai_settings 追加 4 字段：** `image_provider` / `image_base_url` / `image_api_key` / `image_model`
- **新文件：** `src/lib/studio/image-gen.ts`、`src/lib/studio/image-providers/google.ts`、`src/lib/studio/image-providers/openai.ts`、`src/app/api/studio/images/generate/route.ts`、`src/app/api/settings/ai-image/route.ts`、`src/components/studio/image-generate-dialog.tsx`、`src/components/settings/ai-image-settings-dialog.tsx`

### Spec 3：Twitter/X 采集通道

**Spec 文件：** `docs/superpowers/specs/2026-04-06-spec3-twitter-collection-design.md`
**Plan 文件：** `docs/superpowers/plans/2026-04-06-spec3-twitter-collection-plan.md`

#### 架构：Collector 多态派发

之前 `pipeline.ts` 里只用 `rssCollector.collect(...)`。现在引入 `pickCollector(sourceType)` 派发器：

```ts
const collectorMap = {
  'public': rssCollector,
  'custom-rss': rssCollector,
  'twitter-public': twitterPublicCollector,
  'twitter-private': twitterPrivateCollector,
}
```

未来加 YouTube、Newsletter、Reddit collector 时只需加 entry。

- **新文件：** `src/lib/digest/collectors/index.ts`（派发器）

#### Twitter 公开推文通道（今晚可用）

底层复用 RSSHub 的 `/twitter/user/:username` 路由，包装成独立 collector（便于未来 Twitter 特定处理）。

- **新文件：** `src/lib/digest/collectors/twitter-public.ts`
- **快捷添加：** Settings 页源管理区新增「添加 Twitter 源」卡片，输入 `@elonmusk` → 自动生成 `source_configs` 记录 `{ type: 'twitter-public', rssPath: '/twitter/user/elonmusk' }`
- **API：** `POST /api/sources/twitter { username, type: 'public' | 'private' }`

**明早即可用：** 不需要任何登录配置，添加后下次 pipeline 触发就会采集。

#### Twitter 私域时间线通道（代码架子，待你明早授权）

通过 `execFile('agent-browser', ['eval', '--url', 'https://x.com/home', '--script', ..., '--json'])` 调用你全局的 Agent Browser CLI，抓取登录后的 feed DOM，parse 出 tweets。

- **新文件：** `src/lib/digest/collectors/twitter-private.ts`
- **智能 parse：** `parseAgentBrowserOutput` 函数对 JSON 输出和"带前后 log 的 JSON 数组"两种格式做 fallback，兼容性好
- **错误兜底：** 失败时抛明确信息"需要 Chrome + agent-browser + 登录"
- **Health API：** `GET /api/sources/twitter/health` 检测 agent-browser CLI 可用性（UI 用这个做状态徽章）

> ⚠️ **关键：agent-browser CLI 参数是假设性的**，需要你明早验证。见下面「明早待确认事项」。

#### 配置 UI

- **新组件：** `src/components/settings/twitter-source-adder.tsx`（Twitter 图标用 `Bird`，因为 lucide-react 移除了 `Twitter` 图标）
- **集成位置：** Settings 页"我的源"Tab 的源列表下方、"添加自定义 RSS"按钮上方
- **Type 标签扩展：** `TYPE_LABELS` 加入 `twitter-public`（Twitter · 公开）和 `twitter-private`（Twitter · 时间线）两个新标签（sky 色系）
- **懒加载优化：** health API 只在用户切到"我的时间线"tab 时才请求，避免打开 Settings 就触发子进程
